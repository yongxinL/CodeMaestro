# CodeMaestro - Advanced Interactions Reference

**Advanced analytics, estimation, and compliance interactions (Phases 4-5)**

For essential daily interactions, see [INTERACTIONS-CORE.md](INTERACTIONS-CORE.md)

---

## Overview

These advanced interactions are primarily used in:
- **Phase 4 (Verification):** Quality evidence, performance analysis
- **Phase 5 (Release):** Compliance, retrospectives, release decisions

---

## Effort Estimation (Phase F)

### Estimate Project
Get effort estimation for the entire project or milestone.

**Say:** "Estimate effort for this project" / "How long will this take?" / "Project estimation"

**Output includes:**
- Total estimated hours
- Breakdown by milestone
- Resource recommendations
- Risk factors

---

### Estimate with AI
Use AI-powered estimation with complexity analysis.

**Say:** "AI estimate for [feature]" / "Smart estimation" / "Analyze complexity"

**Factors considered:**
- Code complexity
- Integration points
- Testing requirements
- Documentation needs
- Historical data from KB

---

### Compare Estimates
Compare multiple estimation approaches.

**Say:** "Compare estimation methods" / "Which estimate is more accurate?"

---

## Compliance & Ethics (Phase 4-5)

### Check Compliance
Verify regulatory compliance.

**Say:** "Check GDPR compliance" / "Compliance status" / "Are we compliant?"

**Standards supported:**
- GDPR (data privacy)
- SOC2 (security)
- HIPAA (healthcare)
- WCAG (accessibility)
- PCI-DSS (payment)

---

### Ethics Review
Run ethical validation checks.

**Say:** "Ethics review" / "Check for bias" / "Ethical validation"

**Checks:**
- Bias detection in algorithms
- Data privacy compliance
- Accessibility (WCAG)
- Fairness metrics

### Security Audit

**Say:** "Security audit" / "Full security check" / "Vulnerability assessment"

**Includes:**
- Dependency vulnerabilities
- Code security patterns
- Authentication/authorization
- Data encryption
- Secret detection

## Release Management (Phase 5)

### Prepare Release
Generate release preparation checklist.

**Say:** "Prepare release" / "Release checklist" / "Ready for release?"

**Checklist includes:**
- All tests passing
- Security scan clean
- Documentation complete
- Changelog generated
- Version tagged

---

### Generate Changelog
Create changelog from git history.

**Say:** "Generate changelog" / "What changed since last release?" / "Release notes"

---

### GO/NO-GO Decision
Evaluate release readiness.

**Say:** "GO or NO-GO?" / "Release decision" / "Should we release?"

**Factors:**
- Quality gates status
- Security scan results
- Test coverage
- Performance baselines
- Outstanding issues

---

### Framework Cleanup
Remove CodeMaestro files before delivery.

**Say:** "Clean up framework files" / "Prepare for delivery" / "Remove CodeMaestro"

**Modes:**
- "Verify cleanup" - Show what would be removed
- "Apply cleanup" - Remove with confirmation
- "Validate build" - Check artifacts are clean

## Retrospective (Phase 5)

### Lessons Learned
Capture lessons from the project.

**Say:** "Lessons learned" / "What did we learn?" / "Retrospective"

**Categories:**
- What went well
- What could improve
- Process improvements
- Technical discoveries

---

### Token Retrospective
Analyze token usage across project.

**Say:** "Token retrospective" / "How did we do on tokens?" / "Estimation accuracy review"

**Output:**
- Total tokens used
- Variance from estimates
- Phase-by-phase breakdown
- Recommendations for future

---

### Export to Knowledge Base
Export key learnings to organizational KB.

**Say:** "Export to global KB" / "Share learnings" / "Save to organization"
